# Defense Notes

## 1. Краткое описание проекта

SellerHUB — это fullstack-приложение для селлеров маркетплейсов.

Проект помогает продавцу:

| Возможность | Описание |
|---|---|
| Смотреть аналитику | Выручка, заказы, конверсия, товары с низким остатком |
| Управлять товарами | Просмотр товаров, поиск, статус, остатки |
| Работать с отзывами | Просмотр отзывов и ответы покупателям |
| Получать AI-рекомендации | Mock-AI рекомендации по товарам, остаткам и карточкам |
| Использовать Auth | Регистрация, логин, JWT |

---

## 2. Почему выбран такой стек

| Технология | Почему выбрана |
|---|---|
| Node.js | Подходит для backend API |
| Express | Позволяет вручную показать архитектуру без Nest.js |
| TypeScript | Даёт строгую типизацию |
| PostgreSQL | Надёжная реляционная база данных |
| Prisma | Удобные миграции и типизированная работа с БД |
| React | Удобен для frontend dashboard |
| Vite | Быстрая сборка frontend |
| Vitest + Supertest | Проверка API через интеграционные тесты |

---

## 3. Почему не использовался Nest.js

Nest.js не использовался специально, потому что цель проекта — показать собственные архитектурные решения.

В проекте вручную реализованы:

| Слой | Назначение |
|---|---|
| Routes | Описание HTTP endpoints |
| Controllers | Работа с request/response |
| Services | Бизнес-логика |
| Repositories | Работа с базой данных |
| DTO | Типы входных и выходных данных |
| Middlewares | Общая логика, например Auth |

---

## 4. Backend-архитектура

Типовой поток запроса:

```txt
HTTP Request
  ↓
Express Route
  ↓
Controller
  ↓
Service
  ↓
Repository
  ↓
Prisma Client
  ↓
PostgreSQL
```

---

## 5. Frontend-архитектура

```txt
React Page
  ↓
API layer
  ↓
Backend REST API
  ↓
PostgreSQL
```

Frontend разделён на:

| Папка | Назначение |
|---|---|
| `api` | Запросы к backend |
| `components` | Переиспользуемые компоненты |
| `layout` | Sidebar, topbar, общая структура |
| `pages` | Страницы приложения |
| `types` | TypeScript-типы |
| `styles` | Глобальные стили |

---

## 7. Почему AI сделан как mock

AI Assistant реализован как mock-AI, потому что:

| Причина | Польза |
|---|---|
| Не нужен внешний API-ключ | Проект легко запустить |
| Ответы предсказуемые | Удобно тестировать |
| Нет зависимости от интернета | Проект стабилен на защите |
| Есть отдельный service layer | Позже можно заменить mock на реальный AI |
